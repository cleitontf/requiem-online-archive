# Requiem Online — Knowledge Base (Static Archive)

A fully static archive of the **requiem.isnet.ru** community knowledge base for
*Requiem: Memento Mori* (Requiem Online), restored and adapted to run with
**no server-side code, no database, and no login system**.

The original site relied on PHP/AJAX endpoints for things like saved skill
builds, weaponry/fitting tools and quest reward lookups. This archive
recreates the useful parts of that functionality entirely in client-side
JavaScript backed by static JSON/JS data files, so the whole site can be
hosted on GitHub Pages, Netlify, any static file host, or browsed locally.

## What's inside

### Skill Calculator (`/calculator/`)
- All 24 classes/skill trees, with point allocation, tooltips and
  shareable build links (DNA-style encoded URLs).
- Dependency arrows between skills are fully wired up: `template/js/skills_data.js`
  maps each skill to its prerequisite, the matching arrow element on the page,
  and the arrow's visual type — generated by `build_calculator_arrows.py`.

### Monster Database (`/monsters/`)
- Zone overview map linking to 37 zones.
- 866 individual monster pages with stats and drop info.

### Quest Guide (`/quests/`)
- 1,768 individual quest pages with objectives, NPC locations, level, type
  and chain navigation.
- Searchable/filterable/paginated quest list, powered client-side by
  `template/js/quests_data.js` (generated by `build_quests_data.py`).
- Quest reward items (icons, names, quantities) powered by
  `template/js/quest_rewards_data.js` (generated by `build_quest_rewards_data.py`).

### Game Guide (`/guide/`)
- Introduction to the lore of Ethergia and the four playable races (Turan, Bartuk, Kruxena, Xenoa).
- Full class and subclass tree for all 16 subclasses, linked directly to their skill calculators.

## How to run

### Locally
Requires Python 3, no other dependencies:

```bash
python3 serve.py
```

This starts a local server on `http://localhost:8000` and opens it in your
browser automatically. `serve.py` also stubs out the handful of legacy AJAX
endpoints still referenced by the templates so pages don't error out.

### Static hosting (GitHub Pages, Netlify, etc.)
The site is plain HTML/CSS/JS — just deploy the repository root. No build
step is required to browse the site (the generated `template/js/*_data.js`
files are committed).

#### Subpath configuration (`<base href>`)
This repo is published at `https://<user>.github.io/requiem-online-archive/`
(not at the domain root), so every page includes
`<base href="/requiem-online-archive/">` and all internal links/assets are
relative to it. `serve.py` mirrors this prefix locally so links work the
same way under `http://localhost:8000/requiem-online-archive/...`.

If you fork this under a different repo name (or host it at a domain root /
`<user>.github.io`), update the `<base href="...">` value across all
`*.html` files (and the matching `prefix` in `serve.py`'s `translate_path`)
to match your new path — e.g. `<base href="/">` for a root deployment.

## Data generation scripts

The `build_*.py` scripts regenerate the `template/js/*_data.js` files from
source data:

- `build_quests_data.py` reads the 1,768 quest HTML pages under `/quests/`.
- `build_quest_rewards_data.py` reads `Quest.csv` and `ItemAsset.csv` (raw
  game data, not included in the repo) to map quests to reward items.
- `build_calculator_arrows.py` reads each calculator class page and
  `template/images/main.css` to compute the dependency-arrow mapping in
  `skills_data.json`/`skills_data.js`.

These scripts are provided for reproducibility; the generated data files are
already committed, so running them again is only needed if the underlying
HTML/CSS or raw game CSVs change.

## Removed sections

The `/experience/`, `/fitting/` and `/weaponry/` sections from the original
site were removed — they depended on item/weapon databases and AJAX endpoints
not available in this archive. Stub pages are in place at those routes.
See the [Archive Status](about/#status) section for a full breakdown of
what is and isn't preserved.
